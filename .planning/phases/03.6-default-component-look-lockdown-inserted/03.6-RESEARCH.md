# Phase 3.6: Default Component Look Lockdown - Research

**Researched:** 2026-05-24
**Domain:** Renderer theme tokens, preview defaults sync, Tailwind-aligned static palette, golden baseline refresh
**Confidence:** HIGH

## Summary

Phase 3.6 replaces Cora's Phase 2 pastel theme and scattered preview hex literals with a single Tailwind-aligned, slate-neutral default look. Today the renderer and preview diverge on **three separate default layers**: (1) `defaultTheme` pastel tokens + 0.75px edges + default shadows, (2) catalog-component fallbacks in `shared.tsx` (`radius ?? 4`, `CatalogText` default 13px), and (3) preview-only values in `controls/defaults.ts` and per-component overrides in `pack/builtins.tsx`. YAML rendering adds a fourth split: **`box` uses `LayoutBoxNode` (theme path, no radius, 8px `NodeLabel`) while all other components use the catalog path (`nodeCatalogProps` → `CatalogText`, often 12–13px)** — visible in committed goldens (`box-arrows.svg` shows font-size 8 on box, 12 on page, 13 on app).

The locked decisions (D-01–D-13) require a **shared defaults source** consumed by `defaultTheme`, `fontTokens`, `styles.ts`, `measureText.ts`, `Diagram.tsx`, and preview pack/controls. Recommended layout: static Tailwind hex palette module → semantic look tokens → `buildDefaultTheme()` + `catalogDefaultProps(component)` factory. Preview imports the same factory; `builtins.tsx` drops duplicate hex overrides except scenario-specific titles. Golden refresh is intentional (D-13) via `UPDATE_GOLDEN=1 node tests/render-golden.mjs` after all token wiring; ASCII export stays on layouted IR without color/styling (LOOK-06), though larger typography will shift measured node sizes and thus text grid layout — existing text tests assert labels/determinism, not byte-identical snapshots.

**Primary recommendation:** Introduce `renderer/themes/tailwindPalette.ts` + `renderer/themes/lookTokens.ts` + `renderer/themes/componentDefaults.ts` as the single source of truth; rebuild `defaultTheme` from it; route preview defaults through the same module; then refresh all five SVG goldens in one approved commit.

## User Constraints (from CONTEXT.md)

### Locked Decisions

#### Default palette (Tailwind-based)
- **D-01:** Default diagram palette is **neutral professional**: white/near-white fills, dark ink text, subtle gray borders, restrained accent color — not the current pastel renderer theme and not preview UI chrome.
- **D-02:** **Inherit Tailwind CSS color schemes** as the canonical palette vocabulary. Map look tokens to Tailwind scale names and hex values in code. Renderer stays pure SVG — **no runtime Tailwind/CSS dependency** at render time.
- **D-03:** Use the **Tailwind `slate` scale** as the default neutral base for backgrounds, borders, primary text, muted labels, and default surfaces.
- **D-04:** Apply **full Tailwind semantic hue mapping** per built-in component kind (e.g., violet accent/icons, amber decision, rose issue, sky page, emerald app, yellow website, teal shape). Box-like and label nodes stay **slate-neutral** unless their component kind carries a semantic hue.

#### Surface styling (borders, shadows, radii, lines)
- **D-05:** Box-like nodes default to **subtle slate border**: Tailwind slate-300, **1px solid**. Label/icon nodes are borderless or use a slate-200 hairline only where geometry requires definition.
- **D-06:** **No shadow by default** on rendered components. Cast/radial shadow applies only when explicitly set via props/YAML.
- **D-07:** Default corner radius is **medium (~8px, rounded-md feel)** on box-like nodes; decision stays diamond geometry; icon terminals stay circular.
- **D-08:** Default edges and groups use **stronger flow lines**: slate-700 stroke, **2px** default edge width; groups use **slate-400 dashed** boundaries with no fill.

#### Typography and size presets
- **D-09:** Default in-node typography (Noto Sans): **title 12px SemiBold**, **subtitle 10px Regular**.
- **D-10:** Keep existing **sm–xxl size presets** but **tune dimensions** to fit compact typography and medium radius (not replace the preset vocabulary).
- **D-11:** Edge labels: **10px Regular slate-600**. Standalone LabelNode: **11px SemiBold slate-800**, no box fill.
- **D-12:** **Exact preview sync**: preview pack `defaultProps` and inspector control initial values must match renderer resolved output for every built-in component — no duplicate divergent hex values between preview and renderer.

#### Golden baselines
- **D-13:** **Intentionally refresh all SVG golden baselines** to the new locked look in Phase 3.6. Treat this as an approved visual baseline update, not accidental drift. ASCII export must remain unaffected.

### Anti-Patterns to Avoid

- **Per-file hex literals:** Caused current preview/renderer drift — forbidden post-lockdown except in `tailwindPalette.ts`.
- **Preview-only palette in `styles.css`:** Phase 3.5 `--preview-accent` is workbench chrome; do not reuse for diagram tokens [VERIFIED: visual-contract.test.tsx].
- **Theme shadows on by default:** Remove `shadow` from shape tokens and disable `ShapeShadow` for layout box unless explicitly overridden.
- **Split measurement/render font sizes:** Would break PDF overlay alignment and node clipping [VERIFIED: pdf/textOverlay.ts reads theme font sizes].

## Common Pitfalls

### Pitfall 1: Box dual-path visual skew

**What goes wrong:** Preview box looks polished (radius 14, white fill); YAML box stays square, pastel, shadowed, 8px label.  
**Why it happens:** Only `box` uses `LayoutBoxNode`; preview never does.  
**How to avoid:** Apply LOOK radius, no-shadow, slate-neutral fill/stroke in `LayoutBoxNode`; bump `NodeLabel` to 12px; add `rx/ry` to layout rects.  
**Warning signs:** Golden box nodes lack `rx=` attribute; preview/golden side-by-side mismatch.

### Pitfall 2: Font size change shifts layout everywhere

**What goes wrong:** ELK positions, viewBox, PDF text overlay, and text export grid all move when `NODE_FONT_SIZE` 8→12.  
**Why it happens:** `measureText.ts` uses `fontTokens` for node sizing.  
**How to avoid:** Tune `SIZE_PRESETS` + `COMPONENT_MIN_SIZE` together; run full verification suite; refresh goldens once at end (D-13).  
**Warning signs:** PDF tests fail text position assertions; golden diff on first line (viewBox).

### Pitfall 3: Shadow filter defs remain but shadows disabled

**What goes wrong:** SVG still carries `#cora-shadow-blur` filter overhead; accidental shadow if `style.shadow` left on theme shapes.  
**Why it happens:** Legacy `defaultTheme` always set shadow colors.  
**How to avoid:** Omit `shadow` from all default shape tokens; set `shadowBlur: 0`; verify no `filter="url(#cora-shadow-blur)"` in refreshed goldens.  
**Warning signs:** Golden contains `filter="url(#cora-shadow-blur)"` on box nodes.

### Pitfall 4: Preview sync drift returns via builtins overrides

**What goes wrong:** Developer re-adds hex in `builtins.tsx` for "prettier preview."  
**Why it happens:** `builtins.tsx` currently overrides many colors per component.  
**How to avoid:** Limit overrides to `title`/`subtitle` demo strings; import colors only from `componentDefaults`. Add vitest sync test.  
**Warning signs:** `grep '#[0-9A-F]' preview/pack/builtins.tsx` matches more than zero color lines.

### Pitfall 5: Accidental ASCII golden churn

**What goes wrong:** Team interprets LOOK-06 as "text output bytes never change."  
**Why it happens:** Typography affects measurement → layout → text grid.  
**How to avoid:** Assert text tests still pass (labels present, deterministic); do **not** add SVG-style byte goldens for text; document that LOOK-06 means no color/styling in text path.  
**Warning signs:** `render-text.test.ts` label assertions fail after font change.
