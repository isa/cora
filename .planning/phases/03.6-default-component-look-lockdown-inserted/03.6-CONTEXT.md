# Phase 3.6: Default Component Look Lockdown - Context

**Gathered:** 2026-05-23
**Status:** Ready for planning

<domain>
## Phase Boundary

Lock down Cora's built-in component appearance — color palette, typography,
spacing, borders, shadows, line treatments, size presets, and sensible YAML
omission defaults — so rendered diagrams and preview components feel consistent
before Phase 4 interactive canvas editing.

This phase owns:
- canonical default visual language for all built-in nodes, groups, lines,
  labels, and markers (LOOK-01)
- locked default colors with sufficient contrast (LOOK-02)
- locked Noto Sans typography and sizing behavior (LOOK-03)
- normalized sizing, padding, radii, shadows, line widths, and marker sizes
  across renderer and preview (LOOK-04)
- sensible defaults when YAML/component props are omitted (LOOK-05)
- golden baseline refresh protecting the new locked look (LOOK-06)
- documentation of the default look contract (LOOK-07)

This phase does not own:
- preview workbench UI chrome, sidebar typography, or canvas studio styling
  (Phase 3.5 — preview-local only)
- npm package surface, preview distribution, or public API boundaries (Phase 3.7)
- grid snapping semantics (Phase 3.8)
- extension themes or provider icon packs (Phase 5)
- interactive canvas YAML writeback (Phase 4)

</domain>

<decisions>
## Implementation Decisions

### Default palette (Tailwind-based)
- **D-01:** Default diagram palette is **neutral professional**: white/near-white
  fills, dark ink text, subtle gray borders, restrained accent color — not the
  current pastel renderer theme and not preview UI chrome.
- **D-02:** **Inherit Tailwind CSS color schemes** as the canonical palette
  vocabulary. Map look tokens to Tailwind scale names and hex values in code.
  Renderer stays pure SVG — **no runtime Tailwind/CSS dependency** at render time.
- **D-03:** Use the **Tailwind `slate` scale** as the default neutral base for
  backgrounds, borders, primary text, muted labels, and default surfaces.
- **D-04:** Apply **full Tailwind semantic hue mapping** per built-in component
  kind (e.g., violet accent/icons, amber decision, rose issue, sky page, emerald
  app, yellow website, teal shape). Box-like and label nodes stay **slate-neutral**
  unless their component kind carries a semantic hue.

### Surface styling (borders, shadows, radii, lines)
- **D-05:** Box-like nodes default to **subtle slate border**: Tailwind
  slate-300, **1px solid**. Label/icon nodes are borderless or use a slate-200
  hairline only where geometry requires definition.
- **D-06:** **No shadow by default** on rendered components. Cast/radial shadow
  applies only when explicitly set via props/YAML.
- **D-07:** Default corner radius is **medium (~8px, rounded-md feel)** on
  box-like nodes; decision stays diamond geometry; icon terminals stay circular.
- **D-08:** Default edges and groups use **stronger flow lines**: slate-700
  stroke, **2px** default edge width; groups use **slate-400 dashed** boundaries
  with no fill.

### Typography and size presets
- **D-09:** Default in-node typography (Noto Sans): **title 12px SemiBold**,
  **subtitle 10px Regular**.
- **D-10:** Keep existing **sm–xxl size presets** but **tune dimensions** to
  fit compact typography and medium radius (not replace the preset vocabulary).
- **D-11:** Edge labels: **10px Regular slate-600**. Standalone LabelNode:
  **11px SemiBold slate-800**, no box fill.
- **D-12:** **Exact preview sync**: preview pack `defaultProps` and inspector
  control initial values must match renderer resolved output for every built-in
  component — no duplicate divergent hex values between preview and renderer.

### Golden baselines
- **D-13:** **Intentionally refresh all SVG golden baselines** to the new
  locked look in Phase 3.6. Treat this as an approved visual baseline update,
  not accidental drift. ASCII export must remain unaffected.

### Claude's Discretion
- Exact Tailwind shade picks per component kind (e.g., violet-600 vs violet-500)
  and the token file/module layout — constrained by D-02–D-04 and pure SVG output.
- How to implement shared defaults (single module vs theme + component props) —
  constrained by D-12 exact preview sync and existing `themeResolver` seams.
- Specific preset dimension tuning values after compact typography + medium radius.
- Documentation location and format for LOOK-07 (AGENTS.md section, renderer doc,
  or dedicated LOOK contract file).

</decisions>

## Existing Code Insights

### Reusable Assets
- `packages/cora/src/renderer/themes/default.ts` — Central theme token map;
  extend/replace with Tailwind-aligned slate-neutral + semantic hues.
- `packages/cora/src/renderer/components/styles.ts` — `SIZE_PRESETS` and
  `resolveComponentSize()` — tune preset dimensions, keep vocabulary.
- `packages/cora/src/preview/controls/defaults.ts` — Must converge to shared
  locked defaults (D-12).
- `packages/cora/src/preview/pack/builtins.tsx` — Per-component defaultProps
  must match renderer after lockdown.

### Established Patterns
- Renderer components are pure React/SVG; defaults flow through theme resolution
  and component props, not CSS.
- Phase 3.5 preview polish is preview-local; Phase 3.6 sets diagram output
  defaults, not workbench chrome.
- Golden SVG regression protects visual output; Phase 3.6 intentionally refreshes
  all baselines (D-13).
- ASCII export is independent of visual styling — must not change (LOOK-06).

### Integration Points
- Replace pastel `defaultTheme` shapes with Tailwind-aligned token map while
  preserving `resolveNodeStyle()` and `themeResolver` pipeline.
- Introduce or consolidate a **shared defaults source** so preview `defaultProps`,
  control defaults, and renderer theme/component defaults stay identical (D-12).
- Update all kind goldens and representative component output after token lock.
- Align `examples/valid/*.yaml` omitted-prop behavior with polished defaults
  (LOOK-05).

</code_context>

<specifics>
## Specific Ideas

- User direction: **neutral professional** palette, inheriting **all Tailwind
  CSS color schemes** with **neutral/slate as the default base**.
- Semantic component colors should map to Tailwind hues, not ad-hoc one-off hex
  values scattered across preview and renderer.
- Borders stay subtle; no default shadows; medium radius; stronger 2px flow
  lines; compact 12/10 typography inside nodes.
- Preview and renderer must show the same initial/default appearance exactly.

</specifics>
