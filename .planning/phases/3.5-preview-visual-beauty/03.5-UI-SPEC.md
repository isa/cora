# Phase 3.5 — UI Design Contract

> Visual and interaction contract for the `cora preview` workbench polish phase.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none |
| Preset | not applicable |
| Component library | preview-local React components |
| Icon library | existing preview/renderer SVG icons; add no icon package unless planning explicitly justifies it |
| Font | Inter/system UI for preview shell; do not change renderer font defaults in this phase |

This UI-SPEC applies only to the preview application under
`packages/cora/src/preview/`. It must not define canonical renderer component
defaults, renderer component props, or the Phase 3.6 default look contract.

---

## Reference Target

Use the user-attached NodeFlow-style image as the visual target.

Required reference qualities:
- large rounded white side panels on the left and right
- branded left library with prominent `Cora` or `Cora Preview` identity
- `Library / Components` style metadata
- polished diagram-element rows with icon tiles and human labels
- broad center canvas with subtle dotted studio field
- quiet, non-boxy working boundary for the SVG canvas
- split floating canvas tool groups
- reference-style inspector with selected component identity, visual tabs,
  premium controls, swatches, and toggles
- black/white UI with restrained purple accent
- soft shadows, light borders, generous whitespace, and expressive motion

Forbidden direction:
- no landing-page hero
- no marketing composition that hides the workbench
- no one-note teal/slate carryover from the current preview
- no visible generated technical IDs like `node-1` in the primary UI
- no primary toolbar `Labels` control

---

## Spacing Scale

Declared values are multiples of 4:

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon internals, tight inline gaps |
| sm | 8px | Compact row gaps, button icon gaps |
| md | 16px | Default control gaps, panel inner groups |
| lg | 24px | Panel padding, large row rhythm |
| xl | 32px | Canvas and shell gutters |
| 2xl | 48px | Large visual separation inside panels |
| 3xl | 64px | Major desktop composition breathing room |

Exceptions: 6px and 10px may be used only for micro alignment inside controls,
icons, and focus/selection offsets.

Desktop layout constraints:
- side panels should feel like floating soft white slabs rather than hard app
  gutters
- center canvas should keep generous open space even when empty
- controls should never overlap canvas content at 1280px wide or larger

Responsive constraints:
- below 980px, stacked layout is acceptable, but panel content must remain
  readable and no button/control text may overflow its container
- the desktop composition is the primary acceptance target for this phase

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 16px | 400 | 1.45 |
| Label | 12px | 700 | 1.25 |
| Control | 15px | 450 | 1.35 |
| Section heading | 13px | 750 | 1.2 |
| Panel title | 28px | 700 | 1.15 |
| Display | 40px | 700 | 1.05 |

Typography contract:
- use large clean app typography, matching the reference's premium tool feel
- top-left title is prominent and should read as product identity
- section labels are uppercase with letter spacing between `0.08em` and
  `0.12em`; normal paragraph/body text keeps `letter-spacing: 0`
- component names should be human names like `Box`, `Decision`, `Data Source`,
  or existing catalog labels, not internal IDs
- generated technical IDs may remain in DOM attributes or tests but must not be
  visible as primary inspector metadata

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#FBFAF8` | App background and dotted canvas field |
| Surface | `#FFFFFF` | Left/right panels, controls, canvas nodes when framed by preview UI |
| Ink | `#0A0A0A` | Primary text and black action surfaces |
| Muted ink | `#6F6A72` | Metadata labels, secondary body text |
| Border | `#E9E5E1` | Panel/control borders |
| Soft shadow | `rgba(20, 16, 12, 0.10)` | Panel and floating tool depth |
| Accent (10%) | `#6D28D9` | Active tab, selected accent, active swatch, focused states |
| Accent soft | `#EEE5FF` | Icon tile backgrounds and active pills |
| Destructive | `#B42318` | Destructive actions only |

Accent reserved for:
- active segmented/tab state
- active metadata pill
- focus ring when keyboard-visible
- optional selected handle accents, kept minimal
- one or two icon tiles where needed for component identity

Accent must not be applied to every button, every border, or every component
row. Black is the primary action color.

---

## Surface, Depth, and Shape

| Element | Contract |
|---------|----------|
| App background | Warm off-white dotted field across the center canvas area |
| Side panels | White, 24px radius, subtle border, soft shadow |
| Floating tools | White pill/group surfaces, 999px radius where circular/pill, subtle shadow |
| Component rows | 12px radius, white or transparent row surface, icon tile at 36-44px |
| Inspector controls | 12px radius inputs, 1px light border, comfortable vertical padding |
| Swatches | Circular swatches, active ring in purple |
| Selection outline | Thin neutral outline, low noise |
| Handles | Small neutral handles; purple only for sparse selected accent when helpful |

Avoid nested card-in-card composition. Side panels are the only large framed
surfaces; repeated component rows and inspector control groups may have light
row treatments but should not become heavy stacked cards.

---

## Motion Contract

Motion should be expressive but inspection-safe.

| Interaction | Motion |
|-------------|--------|
| Component tile hover | 120-160ms transform or surface lift, no layout shift |
| Component drop/entry | 180-240ms fade/scale into canvas |
| Selection change | 120-180ms outline/handle transition |
| Connection movement | Smooth path transition where feasible; no delayed geometry |
| Panel/tab change | 160-220ms underline or opacity transition |
| Toolbar hover | 120-160ms background/scale feedback |

Respect `prefers-reduced-motion: reduce` by removing transform animations and
using only instant state changes or opacity below 80ms.

## Copywriting Contract

| Element | Copy |
|---------|------|
| App title | `Cora` or `Cora Preview` |
| Left metadata | `Library / Components` |
| Catalog section | `Diagram Elements` |
| Catalog primary action | `Add Component` if it opens/adds a real preview component action; otherwise omit |
| Empty state heading | `Drag components here` |
| Empty state body | Keep subtle; one short sentence max, only if needed |
| Inspector metadata | `Inspector / Attributes` |
| Inspector tabs | `Inspector` and `Style` as visual tabs |
| No selection heading | `No Selection` |
| No selection body | `Select a component to edit its attributes.` |
| Destructive action | `Delete` |

Copy rules:
- no in-app text explaining keyboard shortcuts or design decisions
- no visible generated IDs as primary labels
- use concrete component names and real actions only
- do not invent fake data-source labels unless the selected component actually
  has those props in the preview state

## Component Contracts

### Shell
- `App.tsx` should preserve the three-zone workbench: library, canvas, inspector.
- The visual result should read as a near-match to the reference at desktop
  widths.
- The current top instruction strip should be removed or demoted so it does not
  break the reference composition.

### Library
- `CatalogSidebar.tsx` should present a branded left panel.
- Component rows should use human labels and icon tiles.
- Search may remain but must not dominate the first impression.
- Segmented navigation may be visual-only if there is no real second tab yet.

### Canvas
- `WorkbenchCanvas.tsx` should render on a premium dotted workspace.
- The current bordered SVG should become visually quiet; retain deterministic
  dimensions and viewBox behavior where tests need them.
- Floating tools should be split into smaller groups.
- The visible `Labels` toggle should be removed from primary toolbar chrome.
- Empty state should be calm and sparse; no default sample flow.

### Inspector
- Inspector should show selected-component identity before controls.
- Use an icon tile, selected component title, optional active/status pill, and
  visual `Inspector / Style` tabs.
- Controls remain in one accessible flow unless real tabs are trivial.
- Controls should feel premium: larger inputs, swatches, toggles, and clear
  labels.

---

## Accessibility and Interaction

- Preserve keyboard focus visibility with a 2px purple focus ring or equivalent
  visible state.
- All interactive buttons need accessible labels; icon-only controls need
  `aria-label`.
- Touch/click targets should be at least 40px for primary actions and floating
  tools where space allows.
- Text must not overflow buttons, rows, or inspector fields at supported
  breakpoints.
- Drag/drop behavior must remain usable after styling changes.
- Canvas selection, delete, clear, zoom, and pan behavior must remain covered by
  existing tests or updated equivalent tests.

---

## Verification Contract

Implementation must verify:
- `cd packages/cora && bun run typecheck`
- `cd packages/cora && bun x vitest run tests/preview`
- `cd packages/cora && bun run build`
- browser or screenshot verification of `cora preview --no-open` served locally
  at desktop width

Screenshot acceptance at desktop:
- left panel is a large branded Cora library panel
- center canvas is a premium dotted workspace, not a boxy bordered board
- right inspector has reference-style identity and visual tabs
- palette is black/white plus restrained purple accent
- visible generated IDs like `node-1` are absent from primary UI
- no visible `Labels` toolbar control
- no overlapping text or controls
- floating tools are split into small groups
