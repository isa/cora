# Phase 3.5 — Pattern Map

## Source Files to Modify

| File | Role | Closest Analog / Existing Pattern |
|------|------|-----------------------------------|
| `packages/cora/src/preview/styles.css` | Preview visual system | Existing single source for preview shell, controls, catalog, canvas, inspector, and responsive styling. |
| `packages/cora/src/preview/App.tsx` | Three-zone shell | Already composes catalog, canvas column, and inspector. Preserve this structure. |
| `packages/cora/src/preview/components/CatalogSidebar.tsx` | Left library panel | Existing `catalogItems()` and `filterComponents()` should stay as behavior seams. |
| `packages/cora/src/preview/components/WorkbenchCanvas.tsx` | Canvas and floating tools | Existing toolbar and SVG surface are the primary canvas chrome seam. |
| `packages/cora/src/preview/components/NodePropPanel.tsx` | Node inspector | Existing `groupControls()` separates Content, Appearance, Size. Keep grouping. |
| `packages/cora/src/preview/components/ConnectionPanel.tsx` | Connection inspector | Existing Line/Markers/Routing control groups. Add visual identity without changing controls. |
| `packages/cora/src/preview/components/GroupPanel.tsx` | Group inspector | Existing group control surface. Align visually with other inspector states. |
| `packages/cora/src/preview/components/ControlInput.tsx` | Field rendering | Existing switch by `control.kind`; premium styling should mostly happen via classnames/CSS. |
| `packages/cora/src/preview/components/ui/*` | Primitive controls | Existing local primitives. Prefer extending these over adding a UI dependency. |
| `packages/cora/tests/preview/workbench.test.tsx` | SSR workbench assertions | Update strings and structural assertions for branded library, no `Labels`, no visible generated IDs. |

## Reuse Rules

- Keep preview state, drag, connection, and geometry behavior in
  `state.ts`, `drag.ts`, and `geometry.ts` unchanged unless a visual change
  cannot work without it.
- Prefer CSS variables/tokens at the top of `styles.css` for Phase 3.5 preview
  palette, spacing, radius, and shadows.
- Use existing preview-local UI primitives before introducing dependencies.
- Do not modify `packages/cora/src/renderer/components/` for Phase 3.5 visual
  defaults.
- Do not add fake actions. Any bottom action in the catalog must map to an
  existing preview behavior or be omitted.

## Test Seams

- `renderToStaticMarkup(<App />)` in `workbench.test.tsx` catches structural
  copy and visible chrome.
- `filterComponents()` should keep search behavior testable even if search is
  visually demoted.
- `state.test.ts` and `geometry.test.ts` protect drag, selected item, connection,
  size, and attachment behavior.
- Browser/screenshot verification is required after implementation because
  visual polish cannot be fully proven through SSR markup.
