<objective>
Create the `cora preview` command and package-local Vite preview infrastructure so Phase 3.3 has a runnable browser SPA shell before the workbench features are added.
</objective>

<threat_model>
| Threat | Severity | Mitigation |
|--------|----------|------------|
| T-3.3-01: Vite server serves arbitrary project files outside the package preview root | medium | Configure the preview server with an explicit package-local root and `configFile: false`; do not use the caller's current working directory as the Vite root. |
| T-3.3-02: Tests or CI launch a real browser and hang | low | Provide an `open: false` or equivalent test seam and cover it in CLI/server tests. |
| T-3.3-03: Published package omits preview assets | medium | Add build/package assertions that check deterministic preview output files exist after `bun run build`. |
</threat_model>

  <read_first>

  </read_first>
  <action>
Create `packages/cora/src/cli/commands/preview.ts` and export `registerPreviewCommand(program: Command): void`.

Register it from `packages/cora/src/cli/index.ts` after the existing validate/schema/render registrations.

The command must be named `preview`, take no diagram file argument, and support these options:

- `--host <host>` defaulting to `127.0.0.1`
- `--port <port>` defaulting to `4173`
- `--no-open` to disable opening a browser

Create `packages/cora/src/preview/server.ts` with an exported `startPreviewServer(options)` helper. The helper must import Vite's `createServer`, use `configFile: false`, set an explicit preview root under `packages/cora/src/preview` or the built `dist/preview` equivalent, call `server.listen()`, and return the server plus the actual resolved URL.

The CLI action must call `startPreviewServer`, print the actual localhost URL, and install signal handlers so Ctrl-C closes the server cleanly. Automated tests must be able to call the command/helper with browser opening disabled.
  </action>
  <acceptance_criteria>
- `packages/cora/src/cli/commands/preview.ts` exists and contains `registerPreviewCommand`.
- `packages/cora/src/cli/index.ts` imports and calls `registerPreviewCommand`.
- `packages/cora/src/preview/server.ts` exists and contains `createServer`.
- `packages/cora/src/preview/server.ts` contains `configFile: false`.
- `packages/cora/src/preview/server.ts` exposes an option or parameter that disables browser opening.
- `packages/cora/src/cli/commands/preview.ts` defines `--no-open`.
- `cd packages/cora && bun run typecheck` exits 0.
  </acceptance_criteria>
