# Phase 3.3 — Security

> Per-phase security contract: threat register, accepted risks, and audit trail.

---

## Trust Boundaries

| Boundary | Description | Data Crossing |
|----------|-------------|---------------|
| CLI to local preview server | `cora preview` starts a package-local Vite SPA server from the CLI. | Host/port/open options; no diagram YAML input. |
| Browser preview UI to renderer components | The preview SPA renders built-in component definitions and local React state. | Component props, connection props, drag positions. |
| Package build to installed CLI | Built preview assets are packaged for installed `cora preview` use. | `dist/preview` HTML/CSS/JS assets. |

## Threat Register

| Threat ID | Category | Component | Disposition | Mitigation | Status |
|-----------|----------|-----------|-------------|------------|--------|
| T-3.3-01 | Vite root exposure | Preview server | mitigate | Module-local preview root candidates, `configFile: false`, strict fs; no caller `process.cwd()` preview roots. Evidence: `packages/cora/src/preview/server.ts`. | closed |
| T-3.3-02 | CI browser launch | Preview CLI | mitigate | `--no-open` gates opener calls; injected opener behavior is tested. Evidence: `packages/cora/src/cli/commands/preview.ts`, `packages/cora/tests/cli/preview.test.ts`. | closed |
| T-3.3-03 | Missing preview assets | Package build | mitigate | Package tests and clean-install smoke assert `dist/preview` assets and `preview --help`. Evidence: `packages/cora/tests/preview/package.test.ts`, `packages/cora/tests/smoke/clean-install.sh`. | closed |
| T-3.3-04 | Invalid control props | Preview state controls | mitigate | Connection and node prop updates validate against constrained control definitions and reject invalid values. Evidence: `packages/cora/src/preview/state.ts`, `packages/cora/src/preview/controls/schema.ts`, `packages/cora/tests/preview/state.test.ts`. | closed |
| T-3.3-05 | Accidental public API | Package exports | mitigate | No `./preview` package export is published. Evidence: `packages/cora/package.json`. | closed |
| T-3.3-06 | External pack loading | Preview pack | mitigate | Built-in pack is static/internal; no extension discovery or external pack loading. Evidence: `packages/cora/src/preview/pack/builtins.tsx`, `packages/cora/src/preview/pack/types.ts`. | closed |
| T-3.3-07 | Drag persistence | Preview workbench state | mitigate | Drag and canvas state remain in-memory; no YAML or filesystem persistence imports in preview UI paths. Evidence: `packages/cora/src/preview/drag.ts`, `packages/cora/src/preview/state.ts`. | closed |
| T-3.3-08 | Overlay ambiguity | Attachment overlay | mitigate | Distributed slots, subtle guides, and toggleable labels reduce ambiguity and avoid one-connection-per-side assumptions. Evidence: `packages/cora/src/preview/geometry.ts`, `packages/cora/src/preview/components/AttachmentOverlay.tsx`, `packages/cora/src/preview/components/WorkbenchCanvas.tsx`. | closed |
| T-3.3-09 | Narrow layout | Preview CSS | mitigate | Responsive one-column breakpoint covers narrower screens. Evidence: `packages/cora/src/preview/styles.css`. | closed |
| T-3.3-10 | Requirement traceability | Planning artifacts | mitigate | PREV-01 through PREV-06 are recorded and traced in requirements. Evidence: `.planning/REQUIREMENTS.md`. | closed |
| T-3.3-11 | Installed preview failure | Package install | mitigate | Built and packed preview assets are verified by package tests and clean-install smoke. Evidence: `packages/cora/tests/preview/package.test.ts`, `packages/cora/tests/smoke/clean-install.sh`. | closed |
| T-3.3-12 | Deferred command docs | Documentation | mitigate | Docs describe implemented `cora preview` and keep `cora serve`, `cora ext`, and `cora doctor` deferred. Evidence: `AGENTS.md`, `README.md`. | closed |
