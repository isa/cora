## Tests

### 1. Open Preview Workbench
expected: Start the preview app and open it in the browser. You should see a workbench-first screen with a component catalog on the left, a canvas in the center, canvas controls in the upper-right, and an Attributes inspector on the right.
result: pass

### 2. Drag Components Onto Canvas
expected: Drag components from the catalog onto the canvas. They should appear near the drop location, be selectable, and move smoothly when dragged without the pointer drifting away from the component.
result: pass

### 3. Edit Node Attributes Live
expected: Select a normal node and change visible attributes in the right inspector, including title, subtitle, fill, border, text sizes, and shadow. The selected node should update live on the canvas, with text vertically centered.
result: pass

### 4. Inspect Connections
expected: Place multiple normal nodes and connect them. Selecting a connection should replace the right inspector with connection attributes, show a clear selected state, keep rounded elbows, and render arrowheads at the node edge without line bleed-through.
result: pass

### 5. Use Labels, Icons, And Groups
expected: Add a label node, icon node, label icon node, and group. Labels should attach to connection centers and mask the line, label icons should sit close to the source side with a filled badge, icon nodes should stay tightly square, and groups should be resizable without being treated as node selections.
result: pass

### 6. Use Canvas Tools And Overlay
expected: Toggle anchor labels, zoom in/out, pan with the space bar, delete selected items, and clear the canvas. Anchor guides should stay subtle and distributed when multiple connections use the same side.
result: pass

### 7. Validate Preview Outcome
expected: After trying the core flow, you should be able to judge component appearance and connection behavior in the preview workbench before using the components in diagrams, without the preview requiring or persisting YAML.
result: pass
