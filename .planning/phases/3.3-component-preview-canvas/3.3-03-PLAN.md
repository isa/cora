<objective>
Build the usable workbench UI: catalog/sidebar selection, side-by-side node controls, separate connection controls, draggable two-node canvas, scenario rendering, and designer-facing attachment overlay.
</objective>

<threat_model>
| Threat | Severity | Mitigation |
|--------|----------|------------|
| T-3.3-07: Drag behavior accidentally writes files or YAML | medium | Keep drag state entirely in React/client state and assert no filesystem imports in preview UI modules. |
| T-3.3-08: Overlay labels obscure the node or imply only one connection per side | low | Use computed distributed slots and toggleable labels with subtle side guides. |
| T-3.3-09: UI layout becomes unusable on narrower screens | medium | Implement responsive layout from UI-SPEC and add source assertions for breakpoint classes/styles. |
</threat_model>
