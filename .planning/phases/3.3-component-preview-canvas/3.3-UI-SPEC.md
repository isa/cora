# Phase 3.3 - UI Design Contract

> Visual and interaction contract for the Component Preview Canvas. Generated by gsd-ui-phase and verified inline against the six UI quality dimensions.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none |
| Preset | not applicable |
| Component library | none |
| Icon library | none for preview chrome; renderer component icons come from `cora/renderer/components` |
| Font | Noto Sans, matching the renderer default font family |

Design direction: a focused technical workbench, not a marketing page and not a document gallery. The first screen is the actual preview tool: catalog sidebar, canvas, node prop panels, connection panel, and overlay controls. No hero section, no explanatory feature cards, and no decorative gradient/orb backgrounds.

Renderer purity boundary: preview UI chrome may use React and browser APIs, but renderer components remain pure React/SVG. Preview-specific controls, labels, and manifest metadata live in preview-local source, not under `renderer/components/`.

---

## Layout Contract

Desktop layout is a three-zone workbench:

| Zone | Width | Contents |
|------|-------|----------|
| Catalog sidebar | 240px min, 280px default, 320px max | Pack selector, component search, component family list, primary/secondary node pickers |
| Canvas | Flexible, minimum 520px useful width | Scenario workbench with draggable selected nodes, group context, connection line, attachment overlay |
| Inspector | 360px min, 440px default, 520px max | Side-by-side primary/secondary node prop panels and a separate connection panel |

Responsive behavior:

- At widths below 1024px, inspector moves below the canvas while the catalog remains a left rail if there is at least 768px width.
- At widths below 768px, catalog becomes a top segmented panel, canvas remains first useful content, and node prop panels stack vertically.
- Canvas must maintain stable dimensions with `min-height: 480px` desktop and `min-height: 360px` mobile. Control changes, hover states, and labels must not resize the canvas.
- Side-by-side node panels may scroll internally, but the canvas and catalog must stay visible while editing controls on desktop.

Workbench regions:

- Scenario selector sits above the canvas, not inside the canvas.
- Overlay toggles sit in the canvas top-right corner as compact controls.
- Empty states appear inside their affected region only: sidebar empty search, canvas no selection, or inspector no selected node.
- Do not put UI cards inside other cards. Repeated component list items may be compact rows; only node prop panels and connection panel may use card-like framing.

## Spacing Scale

Declared values must be multiples of 4:

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, anchor label offset, inline color swatch gap |
| sm | 8px | Compact row padding, control label gap, button group gap |
| md | 16px | Panel padding, section spacing, canvas toolbar spacing |
| lg | 24px | Main workbench gutter, inspector group spacing |
| xl | 32px | Wide canvas padding, empty state vertical rhythm |
| 2xl | 48px | Page shell padding on large screens only |
| 3xl | 64px | Not used in Phase 3.3 app chrome |

Exceptions: 1px borders, 2px focus rings, and renderer component coordinates inside SVG previews.

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 14px | 400 | 1.45 |
| Label | 12px | 600 | 1.3 |
| Heading | 16px | 700 | 1.25 |
| Display | 20px | 700 | 1.2 |

Typography rules:

- Do not scale font size with viewport width.
- Letter spacing is `0`.
- Canvas labels and overlay labels use 11px to 12px text and must not obscure node labels or connection endpoints.
- Panel headings stay compact. No hero-scale text in the workbench.
- Long component names truncate with a tooltip in sidebar rows; control labels wrap at word boundaries and must not overlap inputs.

## Component And Control Contract

Catalog sidebar:

- Shows one built-in pack named `built-ins`.
- Lists node components by family: Basic, Web/App, Flow, Issue, Utility.
- Component rows show component label, short family tag, and selected role state: Primary, Secondary, or available.
- Search filters component labels and family tags without changing current canvas state until the user selects a result.

Scenario controls:

- Required scenarios: isolated, connected, grouped, grouped + connected.
- Marker/line variants are selectable starting points for connected scenarios.
- Component-family showcases exist for page, app, issue, and icon-style nodes.
- Scenario switch resets node props and positions to defaults.

Node prop panels:

- Primary and secondary node prop panels are visible side by side on desktop.
- Each panel is generated from typed props for that selected node.
- Controls must prevent invalid values:
  - enums use selects or segmented controls
  - booleans use switches or checkboxes
  - colors use color inputs with visible swatches
  - numbers use bounded sliders or steppers plus numeric input where precision matters
  - size uses a segmented preset selector plus explicit width/height fields when explicit size is selected
- Node panels must show the component id and role, but avoid explanatory prose about how the whole app works.

Connection panel:

- Separate from node prop panels because lines are relationship context, not selected components.
- Controls: line style, stroke color, stroke width, start marker, end marker, and connection mode.
- Connection panel appears only for connected scenarios.
- Line controls update live and do not count as node selection.

Canvas behavior:

- The selected nodes are draggable with pointer and keyboard-accessible nudging.
- Dragging updates line endpoints and attachment slots live.
- Dragging does not persist to YAML and does not write any file.
- Groups are context wrappers. They do not become selected components.

Attachment overlay:

- Shows computed attachment slots for the current scene, not only abstract sides.
- Multiple lines on the same side produce distributed slots labeled `top-1`, `top-2`, `top-3`, etc.
- Subtle side guides show top/right/bottom/left regions.
- Slot labels are toggleable; dots and side guides may remain visible.
- Designer-facing only: no coordinates, path data, routing metadata, or debug mode in Phase 3.3.

## Copywriting Contract

| Element | Copy |
|---------|------|
| Primary CTA | Open Preview |
| Empty canvas heading | Select a primary node |
| Empty canvas body | Pick a node from the built-in pack to start a preview scene. |
| Empty secondary body | Choose a secondary node to inspect connection behavior. |
| Empty search heading | No components found |
| Empty search body | Clear the search or choose another family. |
| Error state | Preview failed to render. Reset the scene or choose another component. |
| Destructive confirmation | Reset scene: reset selected nodes, props, and positions to defaults? |

Microcopy rules:

- Use concrete nouns: node, connection, scenario, overlay, primary, secondary.
- Do not call nodes "components" in selection state labels if it could confuse the node-only rule. Use "Primary node" and "Secondary node".
- Avoid instructional paragraphs in the app chrome. Labels and empty states should be short.
- Button text uses verb + noun where possible: Reset scene, Toggle labels, Select secondary.

---

## Interaction States

Required visible states:

- Default, hover, focus-visible, active, selected, disabled, and loading for catalog rows and controls.
- Primary selected node outline distinct from secondary selected node outline.
- Drag state with cursor and outline feedback.
- Invalid state fallback for any control schema mismatch, even though normal controls prevent invalid input.
- Overlay on/off and labels on/off states.
- Empty states for no primary node, no secondary node in connected scenario, no search results, and preview render error.

Keyboard expectations:

- Tab order: catalog, scenario selector, canvas overlay controls, primary panel, secondary panel, connection panel.
- Arrow keys can nudge the focused selected node when the canvas has focus.
- Escape exits drag/focus mode without resetting props.
- Focus ring uses the accent color and must be visible on light surfaces.

## Registry Safety

| Registry | Blocks Used | Safety Gate |
|----------|-------------|-------------|
| shadcn official | none | not required |
| third-party registry | none | not allowed in Phase 3.3 |

Dependency guidance:

- Do not add a UI component registry dependency for Phase 3.3.
- Do not add an icon library only for preview chrome unless the planner documents a concrete need. Prefer simple text labels, CSS states, and the existing renderer icon slot.
- If a future plan proposes a third-party UI registry or icon package, it must add an explicit verification task covering bundle impact and npm tarball contents.
