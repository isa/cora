## Preview Surface and Navigation

| Option | Description | Selected |
|--------|-------------|----------|
| Component browser first | Sidebar and searchable component list are primary; selecting opens canvas and controls. | |
| Workbench first | Canvas is primary; sidebar and controls support small compositions. | yes |
| Inspector first | Single-component page with examples, props, variants, and attachments. | |
| You decide | Planner chooses based on architecture. | |

**User's choice:** Workbench first.
**Notes:** The user wants to select a component and see connection options with
another chosen component or different components, including group context.
Basic drag and drop is required to inspect different positions and how the
connecting line changes.

| Option | Description | Selected |
|--------|-------------|----------|
| Straight visual connection only | Line endpoint follows dragged components without real routing behavior. | |
| Renderer-like attachment update | Line endpoints snap/recompute against anchors as positions change. | yes |
| Full layout/routing preview | Dragging triggers fuller diagram-style routing behavior. | |

**User's choice:** Renderer-like attachment update.
**Notes:** Dragging should recompute attachment behavior without turning this
phase into the full Phase 4 editor.

| Option | Description | Selected |
|--------|-------------|----------|
| One primary component | One focus component wrapped in contexts. | |
| Two-component comparison | Primary and secondary component explicitly selected. | yes |
| Small scene selection | Any item in the scene can be selected and edited. | |

**User's choice:** Two selected nodes.
**Notes:** The user clarified that "selected components" means nodes only.
Lines/connections and groups should not count as selected components. Maximum
selection is two nodes; both can be dragged and their attributes edited.

## Line Attachment Overlay

| Option | Description | Selected |
|--------|-------------|----------|

**User's choice:** Computed attachment slots plus labels.

## the agent's Discretion

- Exact visual layout, responsive design, and default styling.
- Exact control widgets, as long as invalid prop values are prevented.
- Exact preset names, default props, line variants, and generic group wrapper
  styling.
- Exact dev-server implementation, provided it can seed Phase 4.

## Deferred Ideas

- Saving positions back to YAML belongs to Phase 4.
- Full diagram editing, file watching, unsaved-change handling, and canvas
  persistence belong to Phase 4.
- External pack loading, extension installation, provider/service metadata, and
  icon pack resolution belong to Phase 5.
- Coordinate/path/routing debug overlay is not part of Phase 3.3.
