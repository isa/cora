<objective>
Add the pure text renderer that turns `LayoutedDiagram` into deterministic terminal-readable output, with Unicode as the default and explicit ASCII fallback.
</objective>

<decision_coverage>
- D-01: `renderToText` defaults to Unicode box-drawing characters.
- D-02: `renderToText` consumes layouted node/group/edge coordinates and scales them into a simplified text grid.
- D-03: Tests render all five v1 example kinds through the graph-like core.
- D-04: `RenderToTextOptions.charset = 'ascii'` forces a plain ASCII glyph set.
</decision_coverage>

<threat_model>
| Threat | Severity | Mitigation |
|--------|----------|------------|
| T-3.4-01: Text renderer becomes a second layout engine that diverges from ELK | medium | `renderToText` accepts only `LayoutedDiagram` and scales existing positions; it does not call ELK or parse source diagrams. |
| T-3.4-02: Unicode-only output is unusable in strict ASCII logs | medium | Add explicit `charset: 'ascii'` support and tests for ASCII glyph output. |
| T-3.4-03: Whitespace snapshots become brittle | low | Use small exact assertions only where stable, and source assertions for labels, glyph families, and determinism. |
</threat_model>

  <action>
Create `packages/cora/src/renderer/renderToText.ts` with:

- `export type TextCharset = 'unicode' | 'ascii'`
- `export interface RenderToTextOptions { charset?: TextCharset }`
- `export function renderToText(layouted: LayoutedDiagram, options: RenderToTextOptions = {}): string`

Implement two glyph sets:

- Unicode: use box-drawing characters such as `┌`, `┐`, `└`, `┘`, `─`, `│`, `┼`, and arrows such as `→` / `↓`
- ASCII: use `+`, `-`, `|`, `>`, and `v`

Render a deterministic fixed-width grid from `LayoutedDiagram`:

- groups first, as labeled boxes when `layouted.groups` is present
- nodes as labeled boxes placed near scaled `LayoutedNode.x` / `LayoutedNode.y`
- edges as simplified orthogonal or straight text paths between scaled edge endpoints
- edge labels near the midpoint when `LayoutedEdge.label` exists

Keep the module pure. It must not import from `react`, `react-dom`, `node:fs`, `@resvg/resvg-js`, `pdf-lib`, `yaml`, CLI modules, `window`, or `document`.

Export `renderToText`, `TextCharset`, and `RenderToTextOptions` from `packages/cora/src/renderer/index.ts` and `packages/cora/src/index.ts`.
