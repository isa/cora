## CLI/Error Behavior

| Option | Description | Selected |
|--------|-------------|----------|
| By output extension | `cora render diagram.yaml -o diagram.txt`, matching SVG/PNG/PDF behavior. | ✓ |
| By explicit format flag | `cora render diagram.yaml --output diagram.out --format-output text`, separate from error `--format`. | |
| Both extension and flag | Extension is default path, flag can override for unusual filenames. | |

**User's choice:** By output extension, with additional stdout behavior.
**Notes:** User added that if no output flag is given, the CLI should print the text version to the console.

| Option | Description | Selected |
|--------|-------------|----------|
| Errors only stay JSON | Successful stdout render prints the text diagram; failures with `--format json` print the JSON error array. | ✓ |
| JSON success object | With `--format json`, successful stdout render emits `{ "format": "text", "output": "..." }`. | |
| Disallow stdout render with `--format json` | Require `-o` when machine-readable error mode is requested. | |

**User's choice:** Errors only stay JSON.
**Notes:** We clarified that `--format json` is needed for agents and CI to parse stable failure arrays rather than terminal prose.

| Option | Description | Selected |
|--------|-------------|----------|
| Yes, always print diagram | Piping `cora render diagram.yaml > diagram.txt` works naturally. | ✓ |
| TTY only | Require `-o` when stdout is piped to avoid confusing automation. | |
| Warn on TTY, raw on pipe | Helpful in theory, but risks extra prose. | |

**User's choice:** Yes, always print diagram.
**Notes:** Successful stdout text render should behave the same for TTY and piped stdout.
