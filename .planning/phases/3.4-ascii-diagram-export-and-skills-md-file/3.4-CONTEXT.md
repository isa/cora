# Phase 3.4: ASCII Export + SKILL.md - Context

**Gathered:** 2026-05-23
**Status:** Ready for planning

<domain>
## Phase Boundary

Add a deterministic text export lane for Cora diagrams and create a concise
agent-facing `SKILL.md` guide at the repository root.

This phase owns:
- `cora render diagram.yaml` printing a terminal text diagram to stdout when no
  `-o` is provided
- `cora render diagram.yaml -o diagram.txt` writing the same text export to a
  file
- a terminal-rich Unicode diagram renderer with an explicit pure ASCII fallback
- graph-like text rendering for all v1 diagram kinds using nodes, groups,
  labels, and edges from the existing layouted IR
- root-level `SKILL.md` with triggers, one canonical example, the
  validate/render loop, common fixes, a review checklist, and references to
  `README.md` and `AGENTS.md`

This phase does not own default visual styling, component/icon package surface
decisions, preview distribution changes, extension pack loading, provider icon
resolution, or Phase 4 YAML writeback. Those are covered by Phases 3.5, 3.6,
4, and 5.

</domain>

<decisions>
## Implementation Decisions

### Text Export Output
- **D-01:** The primary text export should optimize for terminal-rich diagrams
  using Unicode box-drawing characters.
- **D-02:** The text renderer should use the existing layouted IR geometry
  loosely. It should preserve rough spatial relationships from ELK while
  simplifying coordinates into a readable text grid.
- **D-03:** First support should cover the graph-like core for all five v1
  diagram kinds: nodes, groups, labels, and edges. It should not try to
  reproduce every kind-specific visual convention in text.
- **D-04:** Include an explicit pure ASCII fallback mode for environments that
  cannot or should not use Unicode. The likely interface is a charset option
  such as `--charset ascii`, but the exact flag name is left to planning as
  long as it is explicit.

### CLI and Error Behavior
- **D-05:** Text output participates in the existing render output-extension
  model: `cora render diagram.yaml -o diagram.txt` writes a text artifact.
- **D-06:** If `cora render diagram.yaml` is called without `-o`, the default
  successful behavior is to print the terminal text diagram to stdout.
- **D-07:** Successful stdout render should print the diagram whether stdout is
  a TTY or piped, so `cora render diagram.yaml > diagram.txt` works naturally.
- **D-08:** `--format json` remains an error-output mode, not an artifact-output
  mode. It exists so agents and CI can parse stable error arrays instead of
  colored human prose.
- **D-09:** On success without `-o`, stdout is the text diagram. On failure with
  `--format json`, stdout must be the existing JSON error array shape.

### SKILL.md
- **D-10:** Create a root-level `SKILL.md`, singular and all caps.
- **D-11:** `SKILL.md` is the quick operating guide for AI agents. `AGENTS.md`
  remains the full contract.
- **D-12:** `SKILL.md` must include trigger language so agents know when to use
  Cora. Example triggers include "architecture diagram", "system diagram",
  "infra diagram", "flowchart", "render YAML diagram", "need SVG/PDF/text
  artifact", and similar diagram-generation requests.
- **D-13:** Keep `SKILL.md` compact: one canonical YAML example, the
  validate/render loop, common repair tactics, and a short review checklist.
- **D-14:** Include references to `README.md` and `AGENTS.md` for deeper reading
  instead of duplicating the full manuals.
- **D-15:** Include `SKILL.md` in the npm package so installed users and
  downstream agents can read the same guide.

### the agent's Discretion
- Exact text-grid scaling, edge-routing simplification, label placement rules,
  and Unicode/ASCII glyph choices are planner/executor decisions, constrained
  by deterministic output and readability.
- Exact charset flag spelling is open, but the fallback must be explicit and
  documented.
- Exact `SKILL.md` wording is open, constrained by the quick-guide role and the
  requirement to include triggers and deeper references.

</decisions>

<code_context>
## Existing Code Insights

### Reusable Assets
- `packages/cora/src/cli/commands/render.ts` already selects SVG/PNG/PDF from
  `-o` extension and centralizes render CLI behavior. Phase 3.4 should add
  `.txt` and stdout text behavior there or through a small helper it calls.
- `packages/cora/src/core/renderPipeline.ts` already computes the layouted IR
  that text export should consume.
- `packages/cora/src/layout-ir.ts` exposes the node, edge, group, point, and
  label information needed to build a simplified text grid.
- Existing examples under `examples/valid/` cover all five v1 diagram kinds and
  should become representative text-export fixtures.
- `packages/cora/tests/smoke/clean-install.sh` is the right place to assert that
  installed package contents include required docs such as `SKILL.md`.

### Established Patterns
- Render failures with `--format json` write a JSON array to stdout and no
  extra prose; preserve this for text export.
- Output type is currently selected by file extension. Keep `.txt` aligned with
  this pattern rather than introducing an artifact-format flag that conflicts
  with the existing `--format` error-output option.
- Renderer components remain pure SVG/React. Text export should be a separate
  renderer over layouted IR and should not leak ASCII concerns into component
  APIs.
- Cora's core value is agent-friendly output and deterministic CI behavior;
  text export should be deterministic and suitable for logs, terminals,
  Markdown, and pull requests.

### Integration Points
- Update `render` command help because `-o` will no longer be required when
  stdout text output is intended.
- Add a text renderer module that accepts `LayoutedDiagram` and charset/options
  and returns a string.
- Add tests for `.txt` output, stdout output, piped/non-TTY behavior where
  practical, Unicode/default charset, ASCII fallback, and JSON error behavior.
- Update `README.md`, `AGENTS.md`, package smoke checks, and root `SKILL.md`.

</code_context>

<specifics>
## Specific Ideas

- Text export should prioritize terminal-rich Unicode box drawing, not a debug
  dump.
- Plain ASCII fallback is still required through an explicit option.
- Missing `-o` should mean "print the text diagram" instead of "error".
- `--format json` is still needed because agents and CI need stable error
  arrays with `code`, `path`, `message`, and `suggestion`.
- `SKILL.md` should be singular/all caps at repo root and should be included in
  the npm package.
- `SKILL.md` should include triggers and one canonical example, then refer to
  `README.md` and `AGENTS.md` for full details.

</specifics>
