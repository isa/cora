<objective>
Integrate text export into `cora render` so `.txt` output and stdout text rendering work while preserving the existing structured error contract.
</objective>

<decision_coverage>
- D-04: CLI exposes explicit ASCII fallback through a charset option.
- D-05: `.txt` output is selected by the existing output extension model.
- D-06: Missing `-o` defaults to stdout text output.
- D-07: Tests cover stdout text behavior.
- D-08: `--format json` is not repurposed for artifact format.
- D-09: JSON failure output is preserved on validation/render errors.
</decision_coverage>

<threat_model>
| Threat | Severity | Mitigation |
|--------|----------|------------|
| T-3.4-04: Relaxing `-o` breaks SVG/PNG/PDF expectations | medium | Missing `-o` selects text stdout only; SVG/PNG/PDF still require explicit output paths. |
| T-3.4-05: Successful text stdout corrupts machine-readable JSON mode | high | Treat `--format json` as error-output mode only; test success and failure paths separately. |
| T-3.4-06: Charset flag accepts invalid values silently | low | Validate charset and fail via Commander error for values other than `unicode` or `ascii`. |
</threat_model>
