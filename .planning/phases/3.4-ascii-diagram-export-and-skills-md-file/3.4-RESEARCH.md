# Phase 3.4: ASCII Export + SKILL.md - Research

## RESEARCH COMPLETE

**Date:** 2026-05-23
**Phase:** 3.4 - ASCII Export + SKILL.md

## Summary

Phase 3.4 can be implemented without adding browser, SVG, PDF, image, or terminal-color dependencies. The right integration point is a new pure text renderer over `LayoutedDiagram`, then a small extension to the existing `cora render` command so `.txt` and missing `-o` select that renderer.

The committed context locks the user-facing behavior:
- Unicode box-drawing is the primary text output.
- An explicit ASCII fallback is required.
- Layout should loosely preserve layouted IR geometry, not mirror SVG exactly.
- `cora render diagram.yaml` prints text to stdout.
- `--format json` remains error-output mode only.
- Root `SKILL.md` is singular/all-caps, compact, trigger-driven, and included in the package.

## Relevant Existing Code

### Render Pipeline

- `packages/cora/src/cli/commands/render.ts` currently owns render command behavior, output extension dispatch, parse/validate/layout, JSON error output, and `-o` enforcement.
- `outputFormat(path)` currently returns only `svg | png | pdf`; `.txt` should be added here.
- The render command currently calls `program.error` if `options.output` is missing. Phase 3.4 should relax this only for stdout text export.
- `--format` already means human/json error output. Do not reuse it for artifact type selection.

### Layouted IR

- `packages/cora/src/layout-ir.ts` provides `LayoutedDiagram`, `LayoutedNode`, `LayoutedEdge`, `LayoutedGroup`, `EdgeLabelPlacement`, and routed edge `points`.
- This is sufficient for text export:
  - node ids and labels from `LayoutedNode`
  - group labels and containment from `LayoutedGroup`
  - edge endpoints, labels, and routed points from `LayoutedEdge`
  - rough geometry from `x`, `y`, `width`, `height`, edge `points`, and diagram `width`/`height`

### Existing Artifact Patterns

- `packages/cora/src/renderer/renderToPNG.ts` is a useful model for a pure artifact renderer with typed options.
- `packages/cora/src/renderer/renderToPDF.ts` is a useful model for consuming layouted IR plus a secondary artifact.
- `packages/cora/src/index.ts` and `packages/cora/src/renderer/index.ts` export renderer helpers; `renderToText` should be exported where package consumers reasonably expect renderer outputs.

### Tests and Smoke

- `packages/cora/tests/render-golden.mjs` shows the fixture set: `box-arrows`, `flowchart`, `microservice`, `infra`, `database`.
- Existing CLI tests use Commander directly for command registration where possible.

## Recommended Implementation Shape

### Text Renderer Module

Create a pure renderer, likely under:

- `packages/cora/src/renderer/renderToText.ts`

Suggested public surface:

- `export type TextCharset = 'unicode' | 'ascii'`
- `export interface RenderToTextOptions { charset?: TextCharset }`
- `export function renderToText(layouted: LayoutedDiagram, options?: RenderToTextOptions): string`

The exact helper split is open, but keep it pure:

- no DOM
- no React
- no SVG parsing
- no terminal color
- no filesystem access
- no browser or image dependencies

### Text Layout Strategy

Use layouted IR loosely:

1. Normalize diagram coordinates into a bounded text grid.
2. Draw groups first as larger boxes when present.
3. Draw nodes as boxes centered near scaled node positions.
4. Draw edges as simplified orthogonal line segments between node centers or nearest sides.
5. Place edge labels near the midpoint when labels exist.
6. Clip or pad deterministically.

The planner/executor may choose exact grid constants, but tests should lock representative output markers rather than overfit every space in a fragile way.

Unicode primary glyphs can use box-drawing characters such as `Ōöī ŌöÉ Ōöö Ōöś ŌöĆ Ōöé Ōö£ Ōöż Ōö¼ Ōö┤ Ōö╝` and arrows such as `ŌåÆ Ōåō`.

ASCII fallback should use plain characters such as `+ - | > v`.

### CLI Integration

Recommended behavior:

- `cora render diagram.yaml -o diagram.txt` writes text.
- `cora render diagram.yaml` writes text to stdout.
- `cora render diagram.yaml > diagram.txt` works.
- `cora render diagram.yaml --charset ascii` prints ASCII text.
- `cora render diagram.yaml -o diagram.txt --charset ascii` writes ASCII text.
- Existing SVG/PNG/PDF output still requires `-o`.
- `--format json` remains for failures. On success, stdout is the artifact text when no `-o` is used.

The render command help must be updated so users understand that `-o` is optional only for text stdout.

### SKILL.md

Create root `SKILL.md`, not `SKILLS.md`.

It should include:

- triggers
- when to use Cora
- one canonical YAML example
- validate/render loop
- output commands for SVG/PNG/PDF/text/stdout
- common fixes keyed to error codes
- review checklist
- links/references to `README.md` and `AGENTS.md`

Package inclusion should be verified after install. Because `packages/cora/package.json` currently has `"files": ["dist", ".npmrc"]`, either the build/package process must copy `SKILL.md` into the package root/dist or the package `files` contract must explicitly include it in a way `npm pack` actually ships.

## Validation Architecture

### Automated Checks

- Unit tests for `renderToText`:
  - minimal diagram includes visible node labels and an edge relation
  - Unicode default output contains box-drawing glyphs
  - ASCII fallback contains `+`, `-`, and `|`
  - all five v1 valid examples render without throwing
  - output is deterministic across repeated calls

- CLI tests for `render`:
  - missing `-o` prints text to stdout and exits 0
  - `-o out.txt` writes text to a file
  - `--charset ascii` changes output away from Unicode glyphs
  - validation failure with `--format json` still prints a JSON error array
  - unsupported output extensions still fail clearly

- Documentation/package tests:
  - root `SKILL.md` exists and contains trigger language
  - package smoke install includes `SKILL.md`
  - README and AGENTS mention text export/stdout behavior

## Risks

| Risk | Severity | Mitigation |
|------|----------|------------|
| Text renderer becomes an alternate layout engine | Medium | Consume `LayoutedDiagram`; use geometry loosely but do not re-run graph layout. |
| stdout text success conflicts with JSON automation | Medium | Keep `--format json` as failure mode only and test success/failure separately. |
| ASCII/Unicode output becomes brittle to whitespace changes | Medium | Prefer source assertions for key glyphs, labels, and edge markers; use exact snapshots only for small minimal fixtures. |
| `SKILL.md` exists in repo but not installed package | High | Add package/smoke assertion against installed `node_modules/cora/SKILL.md`. |
| Preview/package surface scope leaks into Phase 3.4 | Low | Leave preview distribution cleanup to Phase 3.6; only touch package rules needed to include `SKILL.md`. |

## Planning Recommendation

Use four plans:

1. Text renderer module and focused renderer tests.
2. CLI integration for `.txt`, stdout, charset, and JSON failure behavior.
3. `SKILL.md`, README/AGENTS updates, package inclusion, and smoke checks.
4. Requirements trace, all-example text coverage, and full regression verification.
