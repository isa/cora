# Plan 3.1-01 Summary

## Outcome

Phase 3.1 is implemented. The renderer's existing React components now live under
`packages/cora/src/renderer/components/`, `Diagram.tsx` consumes them only through
`./components/index.js`, and the public subpath `cora/renderer/components` is
documented and emitted by the package build.

## Completed Work

- Repaired the inherited `fontkit` metric typing issue in
  `packages/cora/src/core/measureText.ts` without changing the measurement
  formulas.
- Refreshed the five SVG golden fixtures after the inherited Phase 3 drift.
- Moved node, edge, edge-label, edge-decoration, and group implementations under
  `packages/cora/src/renderer/components/`.
- Added stable component prop interfaces:
  `NodeComponentProps`, `EdgeComponentProps`, and `GroupComponentProps`.
- Added the `renderer/components/index.ts` barrel and rewired `Diagram.tsx` to use
  only that barrel.
- Added the package export `./renderer/components` and a dedicated tsdown entry so
  `dist/renderer/components/index.js` and `index.d.ts` are produced.
- Documented the supported v1 renderer contract in `AGENTS.md`.

## Verification

- `cd packages/cora && bun run typecheck`
- `cd packages/cora && UPDATE_GOLDEN=1 node tests/render-golden.mjs`
- `cd packages/cora && bun run build`
- `cd packages/cora && bun run test:golden`
- `cd packages/cora && bun run test`
- Structural checks:
  - `Diagram.tsx` imports from `./components/index.js`
  - no old `./nodes`, `./edges/Arrow`, `./edges/EdgeLabel`, or `./groups/Group`
    imports remain in `Diagram.tsx`
  - no old `NodeProps`, `ArrowProps`, `EdgeLabelProps`, or `GroupProps`
    interfaces remain under `packages/cora/src/renderer`
  - `packages/cora/src/index.ts` does not export renderer components

## Deviations

- `packages/cora/src/renderer/viewBox.ts` was updated even though it was not in
  the original `files_modified` list. This was required because the old
  `edges/decorations.ts` implementation moved under `renderer/components/edges/`.
- `packages/cora/tsdown.config.ts` was updated to add a dedicated component
  build entry. The plan required `dist/renderer/components/index.d.ts`, and the
  existing renderer build did not emit that subpath.

## Follow-Up

Phase 3.2 owns the larger renderer component catalog and normalized style
vocabulary: label/icon/page/app/issue nodes, line components, markers, and richer
style props.
