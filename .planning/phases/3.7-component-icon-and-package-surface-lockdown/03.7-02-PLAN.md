## Phase Goal

**As an** AI agent authoring YAML diagrams, **I want** a built-in set of generic icons I can use via `provider: default` and `service: database` (etc.), **so that** I can create visually meaningful diagrams without installing extensions.

<objective>
Create the default SVG icon set, wire it into the renderer's icon resolution, update semantic validation to accept the built-in provider, and update preview inspector controls to allow icon selection.

Purpose: Establish the v1 built-in icon surface so agents and humans can create icon-bearing diagrams out of the box.
Output: defaultIcons.tsx, updated Diagram.tsx icon resolution, updated errors.ts validation, preview icon controls, icon tests.
</objective>
