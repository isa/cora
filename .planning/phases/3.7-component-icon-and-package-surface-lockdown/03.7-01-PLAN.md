## Phase Goal

**As a** user installing `cora` from npm, **I want** the package to not include preview workbench assets or require vite as a dependency, **so that** install size is minimal and the CLI does not eagerly load development-only infrastructure.

<objective>
Remove the preview workbench from the distributed package surface. Make the preview command development-only by conditionally registering it via dynamic import, relocate vite to devDependencies, and redirect the preview SPA build output to packages/cora/preview-dist/ (outside the published dist/ directory).

Purpose: Shrink the published npm package by ~295KB of preview assets and eliminate vite from the production dependency tree (~30MB+ install weight).
Output: Modified CLI entry point, updated package.json, updated build script, .gitignore entry.
</objective>

Modify packages/cora/src/cli/index.ts:

1. Remove the static top-level import of `registerPreviewCommand` from `'./commands/preview.js'`.

2. Add a development environment detection function. Use filesystem-based detection: check if `packages/cora/src` directory exists relative to the CLI entry point (e.g., `import.meta.dirname` or `__dirname` to resolve the path). This indicates a development (source) checkout vs a published package (only `dist/` exists).

```typescript
import { existsSync } from 'node:fs';
import { resolve } from 'node:path';

function isDevEnvironment(): boolean {
  // In development, the src/ directory exists alongside dist/
  // In published packages, only dist/ ships
  const srcDir = resolve(import.meta.dirname, '..', '..', 'src');
  return existsSync(srcDir);
}
```

3. Replace the static `registerPreviewCommand(program)` call with a conditional dynamic import block:

```typescript
if (isDevEnvironment()) {
  try {
    const { registerPreviewCommand } = await import('./commands/preview.js');
    registerPreviewCommand(program);
  } catch {
    // Preview unavailable — silently skip in production
  }
}
```

4. The top-level function that sets up the CLI (`main()` or the IIFE) must be `async` to support `await import()`. If it is not already async, wrap it.

Do NOT modify `packages/cora/src/cli/commands/preview.ts` — it remains as-is for local development use.
