# Phase 3.7: Component/Icon Package Surface Lockdown - Context

**Gathered:** 2026-05-26
**Status:** Ready for planning

<domain>
## Phase Boundary

Decide and enforce exactly which default components, SVG icon assets, renderer APIs, CLI commands, and built package files ship in the public `cora` package, with the preview workbench retained as a development-phase tool rather than a distributed user-facing feature.

</domain>

<decisions>
## Implementation Decisions

### Preview Command & Dependency Lockdown
- **D-01 (CLI Conditional Registration):** The `preview` CLI command will be conditionally registered in `packages/cora/src/cli/index.ts`. It will be excluded from the CLI help and registration in production. We will dynamically import the preview command registration only when running in development (detected by the existence of the `packages/cora/src` directory).
- **D-02 (Vite Relocation):** `vite` will be moved from `dependencies` to `devDependencies` in `packages/cora/package.json` to reduce the production package size and dependency tree.
- **D-03 (Preview Build Separator):** The Vite build `--outDir` for the preview SPA will be changed to `../../preview-dist` (which outputs to `packages/cora/preview-dist`), placing it outside the `packages/cora/dist` folder. This ensures it is not packed in the published tarball (since `package.json` `files` list only includes `dist`). In local development, the preview server will serve from `preview-dist`. We will add `preview-dist` to `.gitignore`.

### Default SVG Icon Set
- **D-04 (Built-in Provider):** Core will define a built-in provider named `default` (or `cora`) that is always considered installed and does not require extensions.
- **D-05 (Default Icons):** We will ship a default set of generic icons under the `default` provider, including: `server`, `database`, `cloud`, `network`, `user` (alongside the issue status icons `bug`, `warning`, `error`, `stop`).
- **D-06 (YAML Spec Integration):** Nodes in the diagram YAML can specify `provider: default` and `service: <icon-name>` (e.g., `service: database`). Semantic validation in `packages/cora/src/core/errors.ts` will accept `provider: default` as installed and these service names as known, rather than hard-failing.
- **D-07 (Renderer Integration):** The React renderer will dynamically map the `provider: default` + `service: <icon-name>` pair to the corresponding SVG icon component for nodes of component type `icon` and `labelIcon`, instead of hardcoding `WarningIcon`.
- **D-08 (Preview Workbench Controls):** The preview workbench inspector controls will be updated to allow selecting icons from this default set for components that support icons (`BoxNode`, `IconNode`, `LabelIconNode`).

### Public API Export Surface
- **D-09 (Renderer Components Export):** Keep `./renderer/components` as a supported public subpath export in `packages/cora/package.json` so third-party extension and theme developers can import component types.

### Tarball Integrity Checks
- **D-10 (Tarball Smoke Test):** Add an automated integration/smoke test (`packages/cora/tests/smoke/package.test.ts`) that runs `npm pack` (or equivalent), extracts the tarball, and asserts it contains only allowed files (`dist/`, `.npmrc`, `SKILL.md`) and no dev-only assets or Vite configs.

### Claude's Discretion
- The exact visual paths and SVG vector designs for the default icons (`server`, `database`, `cloud`, `network`, `user`).
- The exact structure of the dynamic import statement and conditional checks in `packages/cora/src/cli/index.ts`.
- The exact folder structure and barrel exports of the new default icons module.
- The precise implementation of the package smoke test script.

</decisions>

## Existing Code Insights

### Reusable Assets
- `packages/cora/src/renderer/components/icons.tsx` — Status/issue icons (`BugIcon`, etc.) are defined here. We can add our default icons to a new sibling module or in this file.

### Established Patterns
- `packages/cora/src/renderer/Diagram.tsx` — Custom renderer node mapping. Currently hardcodes `icon={WarningIcon}` for icon nodes. We will refactor this to dynamically resolve the icon component.
- `packages/cora/src/core/errors.ts` — `isExtensionInstalled` and `isKnownService` return false. We will update them to return true for `provider: 'default'` and its default icons.

### Integration Points
- `packages/cora/package.json` — Exports and files array configuration.
- `packages/cora/src/cli/index.ts` — CLI entry point.
