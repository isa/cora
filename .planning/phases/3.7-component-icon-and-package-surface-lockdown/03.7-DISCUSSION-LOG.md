## Preview Command & Dependency Lockdown

| Option | Description | Selected |
|--------|-------------|----------|
| Option 1 | Exclude preview command in production CLI, move Vite to devDependencies, and strip preview assets from build/publish. | ✓ |
| Option 2 | Keep the preview command in the production CLI but throw an error if executed in production. | |

**User's choice:** Option 1 (Recommended)
**Notes:** Excludes the preview CLI command at runtime in production by dynamically testing directory structure, moves Vite to devDependencies, outputs Vite build for preview to `packages/cora/preview-dist` (not included in npm files), and adds it to `.gitignore`.

## Default SVG Icon Set Scope

| Option | Description | Selected |
|--------|-------------|----------|
| Option 1 | Core only packages status/issue icons. No cloud provider icons inside core. Validation hard-fails if extensions are missing. | |
| Option 2 | Ship generic service fallbacks inside core (generic server, db, etc.) and fall back to them instead of hard-failing if extensions are missing. | |
| Option 3 | Support built-in `default` provider to map generic icons (server, database, cloud, network, user) which can be selected via the preview Inspector. | ✓ |

**User's choice:** Option 3 (Custom/User Choice)
**Notes:** Decided that icon sets have two usages: 1) as icons for nodes that support icons, and 2) as nodes themselves. For the first usage, we will add an inspector attribute to pick the icons from the icon sets. For the second usage, we will create node types that will be added to the default component library. A default set of generic icons will be supported under a built-in `default` provider so that other packages can be installed as extensions.
