Add catalog component modules:

- `LabelNode.tsx`: renders text-only label area using `BoxStyleProps`; no required icon.
- `IconNode.tsx`: renders only an SVG icon slot centered in the resolved `size`; no text rendering.
- `LabelIconNode.tsx`: renders a box-like node with icon slot and text.
- `WebsiteNode.tsx`: box-like node with optional text and browser/page affordance using pure SVG primitives.
- `PageNode.tsx`: box-like node with `type`, optional text, `skeletonColorDark`, `skeletonColorLight`, and simple skeleton rows/blocks.
- `AppNode.tsx`: box-like node with optional text and app-window affordance using pure SVG primitives.
- `DecisionNode.tsx`: renders a diamond/decision-style component from `BoxStyleProps`.
- `IssueNode.tsx`: renders a box-like issue node with built-in bug/warning/error/stop icon selection and optional text.
- `ShapeNode.tsx`: low-level escape hatch that accepts `children?: React.ReactNode` in addition to `BoxStyleProps` and renders a pure SVG group at resolved size.

Update `BoxNode.tsx` so it can continue serving the current `NodeComponentProps` render path while also exposing a catalog-friendly component path. If a separate catalog prop component is cleaner, introduce `CatalogBoxNode` internally but keep the exported public name `BoxNode`.

All new components must be pure React/SVG. They must not import YAML/parser/schema/core validation modules or use DOM APIs.

Export every new named component from `components/index.ts`.
