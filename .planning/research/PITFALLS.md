# Domain Pitfalls

**Domain:** Diagram-as-code / visual tooling (Cora)
**Researched:** 2026-05-21

---

## Critical Pitfalls

Mistakes that cause rewrites or block core features.

---

### Pitfall 1: Text Measurement Without a DOM

**What goes wrong:** ELK needs accurate node sizes (width × height) to compute layout. In a headless render context (`renderToStaticMarkup`, CLI), there is no browser DOM — so `getBBox()`, `getBoundingClientRect()`, and `getComputedTextLength()` all return zero or throw. If node sizes are guessed or hardcoded, edge routing and label placement break visually. This is the most common silent failure in headless SVG layout.

**Why it happens:** Projects assume the renderer is responsible for measuring text, or discover too late that ELK's layout pass must happen *before* the React render — so sizes must be computed outside React entirely.

**Consequences:** Labels overflow boxes, edges point at wrong anchor points, groups are sized incorrectly. The canvas interactive view looks fine (browser has a DOM), but headless SVG export looks broken. This difference is maddening to debug.

**Warning signs:**
- Node widths hardcoded to a fixed constant
- ELK receiving `width: 0` / `height: 0` for nodes
- Layout looks correct in `cora serve` but wrong in exported SVG

**Prevention:**
- Use a DOM-free text measurement library (e.g. [Pretext.js](https://pretextjs.dev/) or a font-metrics approach reading OpenType advance widths) to compute label bounding boxes before passing graph to ELK
- Make node-size computation a first-class pure function that runs in both CLI and browser contexts
- Establish a contract: ELK receives fully-sized nodes; the renderer trusts ELK's output positions without measuring again

**Phase:** Core renderer + ELK layout integration (early)

### Pitfall 2: YAML AST Patch Silently Destroying Formatting

**What goes wrong:** The YAML AST save in `cora serve` writes back only positions and labels. But naive implementations using `yaml.set()` / `setIn()` on the AST discard comment attachments, collapse blank lines between sections, and lose indentation variations. An agent's carefully formatted YAML with section comments comes back as dense, comment-free output after the first save. This destroys the agent contract: "save preserves your file."

**Why it happens:** The `eemeli/yaml` library's `set()` operations treat node comments as part of the replaced value and drop them on write. Multiple consecutive blank lines aren't retained in the AST at all.

**Consequences:** Trust breakage with agent workflows. A re-`validate` loop will still pass, but the YAML loses its human/agent readability. Worse, unknown fields added by agents (e.g. `# diagram generated by cora-agent v2`) are silently deleted.

**Warning signs:**
- First save in `cora serve` strips all comments from the source file
- Blank lines between node entries collapse to single newlines
- Fields not in the Cora schema disappear from the file

**Prevention:**
- Transplant comment/spaceBefore properties from old AST nodes to replacement nodes explicitly (not relying on library defaults)
- Use minimal patch: only modify the specific scalar values that changed (positions, label text) — never replace parent collection nodes
- Write a round-trip test: load a richly-commented YAML, save a position change, reload — assert comment presence and blank-line counts

**Phase:** Interactive canvas / YAML patch save

### Pitfall 3: resvg Font Loading Failure in PDF Export

**What goes wrong:** resvg only triggers font discovery when the top-level SVG contains direct `<text>` nodes. If all text is in child SVG elements, referenced `<symbol>` elements, or rendered via `<use>`, resvg silently falls back to a default glyph, producing PDF output with wrong or missing text. This is especially dangerous for icon + label combinations where the label is inside a nested SVG group.

**Why it happens:** resvg's font-loading heuristic is top-level-node based. Nested-SVG text causes `"No match for font-family"` warnings that are easy to miss in CI output.

**Consequences:** PDF export looks correct in browser (Playwright path), but the default resvg path silently produces incorrect or blank labels. Only catches in visual diff review.

**Warning signs:**
- resvg warns `"No match for font-family: ..."` during render
- PDF text uses a fallback monospace glyph while SVG export renders correctly
- Differences visible only at high zoom on PDF

**Prevention:**
- Keep all `<text>` elements at top-level SVG scope, not nested inside child `<svg>` groups
- Add an invisible `<text>` element at the root if the architecture ever requires nested SVGs
- CI: diff rasterized SVG vs PDF for each diagram type on every render, reject on pixel difference above threshold
- Run the resvg render in CI with `RUST_LOG=resvg=warn` and fail on any font-family warning

**Phase:** PDF export / resvg integration

### Pitfall 4: ELK Edge Routing Breaks Across Minor Version Bumps

**What goes wrong:** ELK edge routing behaviour changed silently between versions (notably 0.7.x → 0.8.x). Edges that produced clean orthogonal paths in one version route diagonally or cross nodes in the next. The ELK algorithm properties are sparsely documented; small option changes (e.g. `hierarchyHandling`, `edgeRouting`) can produce dramatically different outputs.

**Why it happens:** ELK is a Java engine compiled to WASM/JS. Its option interactions are complex, and the JS wrapper doesn't validate options — it passes them silently, making misconfiguration invisible.

**Consequences:** Layout regressions that look accidental rather than code-caused. Diagrams that looked great break after a routine `npm update`. Hard to pinpoint because ELK options aren't versioned in the YAML spec.

**Warning signs:**
- `hierarchyHandling` not explicitly set (default changed between versions)
- Edges routing diagonally despite `edgeRouting: ORTHOGONAL`
- Group/compound node edges routing strangely

**Prevention:**
- Pin `elkjs` to an exact version; upgrade deliberately after visual regression testing
- Explicitly set all layout options used (`hierarchyHandling: INCLUDE_CHILDREN`, `edgeRouting`, `nodePlacement.strategy`) — never rely on defaults
- Maintain a golden-image suite of rendered diagrams for each diagram kind; reject PRs that change golden images without explicit approval

**Phase:** ELK layout integration; ongoing in CI

## Moderate Pitfalls

Mistakes that create significant rework but don't block release.

---

### Pitfall 5: Canvas ↔ Headless SVG Rendering Divergence

**What goes wrong:** The React component tree is shared between `cora serve` (browser DOM) and `renderToStaticMarkup` (Node, headless). Browser-specific CSS, `useEffect` animations, `window`/`document` references, or React hooks that only run in browser contexts silently produce different output in each path. The canonical diagram looks polished interactively but flat or broken as an exported artifact.

**Why it happens:** Developers build and preview on the canvas, forget the headless export path, and introduce browser-only code paths. The two paths diverge quietly over time.

**Warning signs:**
- Components using `useLayoutEffect` (throws in SSR)
- Any reference to `window`, `document`, or `getComputedStyle`
- Gradient or filter effects visible in canvas but absent in SVG export

**Prevention:**
- Enforce a "pure SVG component" rule: every renderer component must be a pure function of its props — no effects, no DOM refs, no browser globals
- Run `renderToStaticMarkup` as part of the development cycle (not just CI) to catch divergence immediately
- Use `isomorphic-style` patterns: all visual properties passed as props, not derived from CSS classes at render time

**Phase:** Renderer architecture (foundational)

### Pitfall 6: Extension Version Compatibility Drift

**What goes wrong:** An extension (`aws-theme@1.2.0`) was published against `cora@0.9.x` schema. After `cora@1.0.0` renames a schema field, the extension silently resolves icons to wrong services or fails to apply styles. Because extensions are fetched from a separate git repository at install time, there's no package-manager-enforced compatibility check.

**Why it happens:** Extension manifests declare a `coraVersion: ">=0.9"` but aren't mechanically validated at render time. The lockfile pins the extension version but not the cora-version range check.

**Warning signs:**
- `UNKNOWN_SERVICE` errors appear after a cora upgrade even with lockfile
- Styles applied correctly on install but wrong after `cora ext update`
- Extension tests only run against the cora version at time of PR merge

**Prevention:**
- Encode a `coraSchemaVersion` field in every extension manifest and validate it at extension load time, not just at install time
- Fail loudly (never silently fall back) when a loaded extension declares an incompatible schema version
- `cora ext install` should record the exact cora version in the lockfile, enabling `cora doctor` to detect drift
- `cora-extensions` CI must test each extension against all supported cora minors

**Phase:** Extension system

### Pitfall 7: File-Watch / Unsaved-Canvas Race Condition

**What goes wrong:** `cora serve` watches the source YAML for external changes. An agent runs `cora render` (which doesn't modify the YAML), but then the user's editor auto-saves a reformatted version of the same file. If the file-watch debounce fires while the canvas has unsaved position edits, the guard prompt may appear while the file on disk is already newer — causing the user to "discard" and lose their position work, or "keep" and accumulate a merge conflict in the AST.

**Why it happens:** Debounce-based file watches can fire late. The "external change" event arrives after canvas state has already been mutated. The unsaved-guard logic doesn't distinguish between agent-generated saves and human editor saves.

**Warning signs:**
- File watch debounce is too short (< 300ms) for complex YAML files
- No content-hash comparison between disk state and in-memory AST state (only mtime used)
- Race appears consistently on slower network-mounted filesystems

**Prevention:**
- Compare content hash, not mtime, to detect genuine changes
- The unsaved guard should show a diff of what changed externally so the user can make an informed decision
- The guard must block incoming file-watch reloads while a save is in-flight (not just while edits exist)
- Agent writes via `cora render` should not trigger the unsaved guard (render does not touch the YAML)

**Phase:** Interactive canvas (`cora serve`)

### Pitfall 8: ELK Performance Collapse on Dense Diagrams

**What goes wrong:** Switching ELK's node placement strategy to `NETWORK_SIMPLEX` (for compactness) causes 10× layout slowdown on graphs with 50+ nodes and 100+ edges. Users with large microservice topology or cloud infra diagrams hit 5-10 second layout times in the serve canvas, making drag-to-repin feel unresponsive.

**Why it happens:** ELK's NETWORK_SIMPLEX is not documented as quadratic in node count, but empirically becomes so under heavy partitioning. The default `BRANDES_KOEPF` is dramatically faster.

**Warning signs:**
- Layout time > 1s for diagrams with ~80 nodes
- Layout called synchronously on every position-pin update
- No ELK strategy explicitly set (defaults to slow algorithm in some versions)

**Prevention:**
- Default to `BRANDES_KOEPF`; only expose strategy selection to advanced users
- Run ELK in a Web Worker in `cora serve` to avoid blocking the canvas
- Cache layout result keyed on graph topology hash; only re-layout on structural changes, not position-pin changes
- Add a layout-time benchmark with a 100-node graph to CI

**Phase:** ELK layout integration; `cora serve` canvas

## Minor Pitfalls

Annoyances and polish issues that surface after initial release.

---

### Pitfall 9: SVG viewBox Cropping

**What goes wrong:** `renderToStaticMarkup` produces an SVG with a `viewBox` tightly fit to the ELK-computed layout bounding box. Nodes near the edge have their drop shadows, stroke widths, or label overflows clipped. The cropped artifact looks amateur regardless of the diagram quality.

**Warning signs:** Node shadows or strokes cut off at the SVG boundary in exported files.

**Prevention:** Add fixed padding (e.g. 24px) to all four sides of the viewBox after ELK layout. Make padding a named constant in the renderer, not a magic number.

**Phase:** SVG export

---

### Pitfall 10: Font Not Embedded in SVG Export

**What goes wrong:** The exported SVG references a web font by name (e.g. `font-family: "Inter"`). When the SVG is opened in a context without that font (another machine, GitHub preview, Slack paste), text falls back to the system serif — destroying the visual polish.

**Warning signs:** SVG looks correct in the browser where you rendered it, but wrong when shared or embedded.

**Prevention:** Either use a system font stack (`-apple-system, BlinkMacSystemFont, "Segoe UI", system-ui, sans-serif`) for the default theme, or embed the font as a `<style>` block with a base64 `@font-face` in the exported SVG. Document the tradeoff (file size vs portability).

**Phase:** Default theme / SVG export

---

### Pitfall 11: `renderToStaticMarkup` SVG Namespace Warnings

**What goes wrong:** React warns (or corrupts output) when SVG elements like `<linearGradient>`, `<feDropShadow>`, or `<clipPath>` appear outside a parent `<svg>` tag during server rendering. React applies HTML namespace rules to orphaned SVG elements, which breaks the element names in the output.

**Warning signs:** `Warning: <linearGradient> is not a valid SVG element outside of <svg>` in Node.js output.

**Prevention:** Always render SVG components within a root `<svg>` element. Never render SVG sub-components in isolation via `renderToStaticMarkup` without the `<svg>` wrapper.

**Phase:** Renderer architecture

---

### Pitfall 12: Agent Error Messages That Are Human-Only

**What goes wrong:** `cora validate` returns ANSI-colored, emoji-decorated terminal output that looks great in a terminal but is noise when consumed by an agent via subprocess stdout capture. Agents try to parse the colorized output, fail, and either hallucinate the error or treat the whole command as succeeded.

**Warning signs:** `cora validate` output contains ANSI escape codes when piped; no `--format json` flag; error messages use English prose instead of stable codes.

**Prevention:**
- `cora validate --format json` must be the agent-safe path; produce structured errors with stable codes (`UNKNOWN_SERVICE`, `MISSING_EDGE_TARGET`), paths, and line numbers
- Default to human output only when stdout is a TTY (`process.stdout.isTTY`); pipe automatically gets plain text or JSON
- AGENTS.md must document the exact JSON output schema and all error codes

**Phase:** CLI / agent contract (foundational)

## Phase-Specific Warnings

| Phase Topic | Likely Pitfall | Mitigation |
|-------------|---------------|------------|
| ELK layout integration | Text measurement in headless (Pitfall 1) | Resolve DOM-free measurement library before writing layout code |
| ELK layout integration | Edge routing regression on version bump (Pitfall 4) | Pin version; golden image CI from day one |
| SVG renderer | Canvas ↔ headless divergence (Pitfall 5) | Enforce pure-function component rule at architecture stage |
| SVG renderer | viewBox cropping (Pitfall 9) | Add padding constant before first export test |
| SVG renderer | Font not embedded (Pitfall 10) | Decide system-font vs embedded-font before theme work |
| SVG renderer | namespace warnings (Pitfall 11) | Never render SVG components outside `<svg>` wrapper |
| PDF export | resvg font loading (Pitfall 3) | Keep all `<text>` at top-level SVG scope |
| Interactive canvas | Canvas ↔ headless divergence (Pitfall 5) | Same root cause; shared prevention |
| Interactive canvas | File-watch race condition (Pitfall 7) | Use content hash, not mtime |
| Interactive canvas | YAML AST patch destroying formatting (Pitfall 2) | Build round-trip test before implementing patch save |
| Interactive canvas | ELK performance on large graphs (Pitfall 8) | Worker thread + layout caching |
| Extension system | Extension version drift (Pitfall 6) | Schema version in manifest + load-time validation |
| CLI / agent contract | Agent-hostile error output (Pitfall 12) | TTY detection + `--format json` in first CLI iteration |
